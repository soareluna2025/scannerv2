── ENDPOINT-URI PRINCIPALE (/api/*) ─────────────────────────────────────────

GET /api/football
  Sursă:    API-Football (live=all) + football-data.org (backup)
  Ieșire:   { response: [matches...] } — format API-Football
  Filtre:   ALLOWED_LEAGUE_IDS, WOMEN_RE, LOWER_DIV_RE, YOUTH_RE
  Frecvență: la cerere din frontend (loadLive interval ~30s)
  Note:     Enrichment cache în memorie per pereche home-away (max 5 concurrent)

GET /api/today
  Sursă:    PostgreSQL (fixtures table) → fallback API-Football dacă <5 meciuri în DB
  Ieșire:   { response: [matches...] }
  Filtre:   ALLOWED_LEAGUE_IDS + all 3 regex filters
  Frecvență: la cerere din frontend (loadPM interval)
  Side effect: upsertează meciuri NS în fixtures table (din fallback API)

GET /api/enrich?h=&a=&fid=&lgid=&elapsed=&hg=&ag=&soth=&sota=&ref=&br=
  Sursă:    DB primary (form din fixtures_history, h2h, odds) → fallback API-Football
  Ieșire:   calcul complet Poisson + EV + Confidence Score 7 straturi + Kelly

  Side effect: INSERT în predictions (ON CONFLICT DO NOTHING)
              fetchAndStoreInjuries (fire-and-forget)

  Frecvență: la click detalii meci / modal
  Side effect: INSERT în predictions (ON CONFLICT DO NOTHING)

── CRON ENDPOINT-URI (/api/cron/*) ──────────────────────────────────────────

POST /api/cron/prematch-enrichment
  Frecvență: */5 minute (crontab)
  Ce face: 7 etape de colectare date pre-meci per fixture (24h window)
  Etape bazate pe ore rămase până la kickoff:
    Stage 1: 23-25h → fixture, venues, coaches, h2h, injuries, squads, standings,
                       predictions, odds, team-stats
    Stage 2: 11-13h → similar + actualizare odds
    Stage 3: 5-7h   → actualizare injuries, odds fresh
    Stage 4: 2.5-3.5h → odds finali, weather
    Stage 5: 1.5-2.5h → lineup predictions
    Stage 6: 45-90min → lineup confirmat, odds finali
    Stage 7: 0-45min  → date complete finale
  Salvează în: prematch_data (JSONB per fixture/stage/data_type)
              prematch_enrichment_log (tracking per fixture/stage)

POST /api/cron/collect-daily
  Frecvență: zilnic 06:00 (crontab)
  Ce face:
    1. Fetch standings per ligă (SEASON = year-1 dacă mai < august)
    2. Upsert leagues table
    3. Upsert teams table (ÎNAINTE de standings — FK constraint)
    4. Upsert standings table
    5. Calcul form_stats via SQL batch din fixtures_history (last5, avg_scored/conceded)
  Ligi procesate: ~165 (toate din ALLOWED_LEAGUE_IDS)
  API req: ~165 req (1 per ligă)

POST /api/cron/collect-finished
  Frecvență: zilnic 23:00 (crontab)
  Ce face:
    1. Fetch meciuri FT din ziua curentă (multiple ligi)
    2. Upsert în fixtures_history
    3. Per meci FT: collect player_stats, match_stats, match_events, odds
  API req: ~100-300 per rulare (depinde de meciuri terminate azi)

POST /api/cron/league-stats
  Frecvență: zilnic 04:00 (crontab) — ZERO apeluri API
  Ce face: calculează statistici per ligă din fixtures_history + match_stats
           → upsert în league_stats
  Crează tabel dacă nu există (CREATE TABLE IF NOT EXISTS)

POST /api/cron/referee-stats
  Frecvență: zilnic 04:00 (crontab)
  Ce face:
    1. Fetch 33 zile de fixtures din API-Football (30 zile înapoi + 3 zile viitor)
    2. Upsert FT meciuri în fixtures_history (cu referee)
    3. Calcul statistici per arbitru din fixtures_history + match_stats + match_events
    4. Upsert în referee_stats
  Crează tabel dacă nu există + ALTER TABLE fixtures_history ADD COLUMN IF NOT EXISTS referee

POST /api/cron/scan
  Frecvență: via scanner.js intern (background loop — NU crontab)
  Ce face: fetch meciuri LIVE, calcul NGP/markets/GG/Over, upsert match_snapshots
           salvare live_stats, h2h, form_stats, alerts

── SCANNER INTERN (scanner.js — background process) ─────────────────────────

startScanner() — pornit din server.js la startup

6. ALGORITME PRINCIPALE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

A. POISSON 6×6 (calc-utils.js → calcPoisson6x6)
   Matrix 6×6 scoruri (0-0 până la 5-5):
     P(i,j) = Poisson(λH, i) × Poisson(λA, j)
   Derivate:
     homeWin = ΣP(i>j), draw = ΣP(i=j), awayWin = ΣP(i<j) → normalizate la 100%
     over15 = ΣP(i+j≥2), over25 = ΣP(i+j≥3), ggProb = ΣP(i>0 AND j>0)

   Lambda calcul (enrich.js):
     λH = (homeAvgScored + awayAvgConceded) / 2   ← cross-product model corect
     λA = (awayAvgScored + homeAvgConceded) / 2
   Dacă date insuficiente: lgHome (avg_home_goals din league_stats) sau 1.2

   Dynamic lambda (mecuri live):
     minutesLeft = 90 - elapsed
     intensityFactor = 1 + min(SOT/25*elapsed*90⁻¹, 0.4)
     λ_remaining = λBase × (minutesLeft/90) × intensityFactor
     λ_dynamic = currentGoals + λ_remaining

B. EXPECTED VALUE (calc-utils.js → calcEV)
   impliedH = 1/cotaH, impliedD = 1/cotaD, impliedA = 1/cotaA
   totalImplied = impliedH + impliedD + impliedA
   evHome = (homeWin/100) - (impliedH / totalImplied)
   evOver15 = (over15Prob/100) - (1/cotaOver15)
   evGG     = (ggProb/100) - (1/cotaGG)

   Kelly:
     kelly = min(bankroll × edge × 0.5, bankroll × 0.04)   dacă edge > 4%
     altfel: kelly = 0

C. CONFIDENCE SCORE 7 STRATURI (enrich.js → calcConfidence)

   Layer 1 — Poisson over15Prob (22%)
   Layer 2 — Formă recentă: min(100, (homeAvg+awayAvg)/3.5*100) (20%)
   Layer 3 — H2H over15% (10%)
   Layer 4 — Live xG/SOT/DA: xg×25 + sot×3 + da×0.5 (15%)
              → dacă SOT=0 după min10: score4 = 20 (penalizare)
   Layer 5 — EV: best market EV×300 (8%)
   Layer 6 — Consistență: (layere>60 / 5) × 100 (5%)
   Layer 7 — Puterea echipei din player_stats (20%) — activ dacă ≥10 jucători/echipă

   Penalizări runtime:
     elapsed < 15min: score × 0.85
     elapsed 15-30min + SOT=0: score × 0.80
     elapsed ≥ 45min + SOT=0: score - 20
     yellow_cards ≥ 2: score - 10
     injuries ≥ 3/5/8: score - 5/10/15

D. TEAM STRENGTH / LAYER 7 (enrich.js → getTeamStrengths)
   Din ultimele 110 rânduri player_stats per echipă:
     teamStrength = avgRating/10×100×0.35
                  + min(100, goalsPerGame×35)×0.25
                  + avgPassAcc×0.20
                  + min(100, avgSOT×12)×0.10
                  + min(100, topScorer×20)×0.10
   Activ doar dacă ≥10 rows per echipă.

E. PLAYER SCORE (calc-utils.js → calcPlayerScore)
   playerScore = ratingNorm×0.35 + goalsScore×0.20 + assistScore×0.15
               + passScore×0.20 + shotScore×0.10
   Scala 0-100 per jucător.

F. NGP — Next Goal Probability (scan.js / scanner.js)
   ngp = SOT×4.5 + ShotsOff×1.5 + DangerousAttacks×0.3
        + Corners×2.0 + Goals×8 - (elapsed/90)×15
   Normalizat 0-100%. Alertă dacă NGP > 85 sau over15 > 82.

G. MONTE CARLO (monte-carlo.js → runSimulation)
   10.000 simulări:
     - Goluri adiționale = poissonRandom(λH) + poissonRandom(λA)
     - Scor final = currentGoals + adiționale
   Output: homeWin%, draw%, awayWin%, over05/15/25/35%, GG%, scoreDistribution

H. ELO (elo.js → calcElo)
   K=32, rating inițial=1500
   expH = 1/(1 + 10^((eloA-eloH)/400))
   update = eloH + K × (actual - expected)
   homeWinProb = 1/(1 + 10^(-eloDiff/400))
   Folosit în: simulate.js (lambda ajustare via eloFactor)

I. MATCH MOMENTUM (match-momentum.js → calcMomentum)
   homeScore = SOT×15 + DA×3 + possession×0.5 + corners×5 + xG×20
   score = (homeScore-awayScore)/(homeScore+awayScore) × 100  → [-100, 100]

J. GENERATOR SCORING (index.html → g2Score)
   Lambda cross-product: lamH = (home_avg_scored + away_avg_conceded) / 2
   Categorii:
     home/away → g2Over(lam, 0.5) cu h2h si live context
     goals     → g2Over(lamH+lamA, threshold) cu h2h si ref_stats
     gg        → compH=(pHs×pAc)/100, compA=(pAs×pHc)/100 + h2h_GG
     cards     → defIntensity factor + ref.avg_yellow + lg.avg_yellow
     corners   → attFactor + ref.avg_corners + lg.avg_corners

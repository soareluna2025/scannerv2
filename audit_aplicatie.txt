--- api/football.js ---
GET /api/football
Primește: nimic (fără parametri)
Face:
  1. fetch API-Football /fixtures?live=all → filtrare status live (1H/HT/2H/ET/BT/P/SUSP/INT/LIVE)
  2. fetch football-data.org /matches?status=IN_PLAY,PAUSED,HALFTIME (dacă FOOTBALL_DATA_KEY)
  3. Merge: AF primar, FD completează golurile
  4. Filtrare: whitelist ALLOWED_LEAGUE_IDS (doar AF), filtru feminin+div3+
  5. Îmbogățire opțională (max 5 meciuri): H2H + liveStats din calcFeatures
  6. Cache îmbogățire în Map (key: homeId-awayId)
Returnează: { response: [...meciuri merged și filtrate] }

--- api/today.js ---
GET /api/today
Cache: 120s (s-maxage)
Face:
  SURSA 1 (Supabase-first):
    - Citește fixtures din Supabase (kickoff azi+24h, status NS)
    - Dacă ≥ 5 rezultate: aplică whitelist → fetch league names → transformare format AF
  SURSA 2 (fallback API-Football):
    - /fixtures?date={azi}&status=NS + /fixtures?date={mâine}&status=NS
    - Deduplicare + fereastră 24h
  Filtrare: whitelist ligi + feminin + div3+
Returnează: { response: [...], _debug: { source: 'supabase'|'api-football', ... } }

--- api/enrich.js ---

Face:
  BATCH 1 (paralel):
    - Supabase form_stats per echipă (limit 10)
    - Supabase h2h bidirectional (limit 10)
    - Supabase odds (bookmaker_id=8, dacă fid prezent)

  BATCH 2 (fallback API-Football unde Supabase < 3 rânduri):
    - /fixtures?team={h}&last=20&status=FT
    - /fixtures?team={a}&last=20&status=FT
    - /fixtures/headtohead?h2h={h}-{a}&last=10

  Calcule: calcPoisson → calcEV → calcConfidence (7 straturi)
  Lambda dinamic dacă elapsed > 0
  Salvare fire-and-forget în Supabase.predictions
Returnează: { lambdaHome, lambdaAway, lambdaTotal, over15Prob, over25Prob, ggProb,

--- BUGS DE DATE ---

3. [BUG] match_events fără upsert — collect-finished.js face INSERT pur în match_events:
   ```javascript
   await fetch(`${sbUrl}/rest/v1/match_events`, { method: 'POST', ... })
   ```
   Nu are ?on_conflict= și nu are resolution=merge-duplicates.
   Dacă cronul rulează de două ori pe același fixture (sau manual), evenimentele
   se duplică în baza de date.

4. [BUG] player_stats upsert fără on_conflict în URL:
   În api/players.js (backfill) și api/cron/collect-finished.js:
   ```javascript
   await fetch(`${sbUrl}/rest/v1/player_stats`, { method: 'POST',
     headers: { 'Prefer': 'resolution=merge-duplicates,return=minimal' }, ... })
   ```
   PostgREST necesită ?on_conflict=player_id,fixture_id în URL pentru
   resolution=merge-duplicates. Fără aceasta, conflictele nu se rezolvă corect
   și pot apărea duplicate sau erori silențioase.
   (Notă: collect-finished.js are corect ?on_conflict= pentru match_stats și odds,
   dar NU pentru player_stats)

5. [BUG] scan.js comentariu incorect: "// Vercel Cron Job — runs every minute"
   Dar vercel.json îl rulează ZILNIC la 00:00 (schedule: "0 0 * * *").
   NGP și alertele live se calculează o singură dată pe zi, nu în timp real.
   Aceasta înseamnă că funcționalitatea live din frontend (refresh 30s) calculează
   NGP în JavaScript pe client, nu pe server.
